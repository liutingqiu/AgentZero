## 需求

在 `upload_handler` 函数中添加服务端文件大小上限和文件类型白名单校验。

### 实现要点

1. **在 upload_handler 函数头部定义以下常量**：

   ```python
   MAX_FILE_SIZE = 10 * 1024 * 1024  # 10MB
   ALLOWED_EXTENSIONS = {
       '.txt', '.md', '.json', '.py', '.js', '.ts', '.html', '.css', '.scss',
       '.png', '.jpg', '.jpeg', '.gif', '.webp', '.svg', '.ico', '.bmp',
       '.pdf', '.doc', '.docx', '.xlsx', '.xls', '.ppt', '.pptx', '.csv',
       '.zip', '.tar', '.gz', '.7z', '.rar',
       '.mp3', '.mp4', '.wav', '.flac', '.ogg', '.webm', '.mov', '.avi',
       '.log', '.yaml', '.yml', '.toml', '.ini', '.cfg', '.conf',
       '.xml', '.sql', '.sh', '.bat', '.ps1',
       '.mdx', '.tex', '.rst',
       '.woff', '.woff2', '.ttf', '.eot',
   }
   ```

2. **在处理文件写入之前添加校验**：

   a. **文件扩展名校验**：
      ```python
      ext = os.path.splitext(filename)[1].lower()
      if ext not in ALLOWED_EXTENSIONS:
          return web.json_response({'error': f'文件类型不允许: {ext}'}, status=400)
      ```

   b. **文件大小校验**：
      在逐块读取文件内容时，累计已读取的字节数。如果超过 `MAX_FILE_SIZE`，立即清理已保存的部分并返回 400 错误。

3. **已有防护保留**：`os.path.basename()` 路径穿越防护要保持不变。

---

## 验收标准

1. 上传 `.exe` 文件 → 返回 HTTP 400 + `{"error": "文件类型不允许: .exe"}`
2. 上传 11MB 的大文件 → 返回 HTTP 400 + `{"error": "文件过大，最大允许 10MB"}`
3. 上传合法的 `.txt` 文件（小于 10MB）→ 返回 HTTP 200，文件正常保存
4. 上传合法的 `.png` 图片 → 返回 HTTP 200，文件正常保存
