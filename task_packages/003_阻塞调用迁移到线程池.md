## 需求

将 SSE 流式处理 handler 中直接调用的同步阻塞函数（`handle_message`、`orch.run`）包装到线程池执行，防止阻塞 aiohttp 事件循环。

### 背景

当前 `ChatStreamHandler` 中直接调用 `handle_message(message)`，这是一个同步函数，执行时可能需要数秒到数十秒（调用外部 LLM API）。在 aiohttp 的事件循环中执行同步阻塞调用会导致**所有并发请求都被阻塞**——一个慢请求会拖慢整个服务。

### 实现要点

1. **ChatStreamHandler 改造**（约第 167-195 行）：

   在 `ChatStreamHandler` 类中：
   - 在 `_send` 方法中，找到调用 `handle_message` 的地方
   - 改为使用 `asyncio.get_event_loop().run_in_executor(None, handle_message, message)`
   - `handle_message` 返回的结果类型不变，后续处理逻辑不变

2. **CollabStreamHandler 改造**（约第 229-268 行）：

   同样的方式处理 `orch.run(message, wm=wm)`：
   - 改为 `await asyncio.get_event_loop().run_in_executor(None, orch.run, message, wm)`
   - 或者直接使用 `loop.run_in_executor(None, orch.run, message, wm)`

3. **注意事项**：
   - 只需修改调用方式，不修改 `handle_message` 或 `orch.run` 的内部逻辑
   - 保持 await 语义不变：在 Inspector 执行后被 await，确保后续代码在任务完成后执行
   - 保持异常处理不变：try/except 逻辑保持原样

### 改动示例（伪代码）

```python
# 改动前
result = handle_message(message)

# 改动后
loop = asyncio.get_event_loop()
result = await loop.run_in_executor(None, handle_message, message)
```
