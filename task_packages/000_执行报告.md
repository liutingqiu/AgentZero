### 001 · CORS 白名单限制

**目标**：将 CORS 从 `*` 通配改为白名单校验。

**实现**：

- `_cors_headers(request)` 函数签名扩展，支持传入 `request` 参数
- 从 `Origin` 请求头提取来源，与白名单 `['http://127.0.0.1:5052', 'http://localhost:5052']` 比对
- 匹配 → 返回该 Origin；不匹配 → 不设置 `Access-Control-Allow-Origin` 头
- 无 request 时保留 `*` 兜底（用于 OPTIONS 预检等场景）

**验证**：`_cors_headers()` 函数已部署到所有 API 路由 handler [server.py:43-63]

### 008 · 后端看板 CRUD 接口

**目标**：为看板模块暴露 POST/PUT/DELETE HTTP 接口，实现完整 CRUD。

**实现**：

| 端点 | 方法 | Handler | 认证 | 说明 |
|------|:--:|------|:--:|------|
| `/api/kanban` | POST | `KanbanHandler.post` | ✅ | 创建任务，要求 `title` 字段 |
| `/api/kanban/{id}` | PUT | `KanbanTaskHandler.put` | ✅ | 更新任务状态，校验状态值 |
| `/api/kanban/{id}` | DELETE | `KanbanTaskHandler.delete` | ✅ | 删除任务，先查存在 |

**适配要点**（任务包假设 vs 实际代码）：

| 维度 | 任务包假设 | 实际代码 | 处理方式 |
|------|----------|---------|---------|
| 路由模式 | `app.router.add_post` | aiohttp `RouteTableDef` + View 类 | 沿用 View 模式，`KanbanHandler` 新增 `post()` + 新建 `KanbanTaskHandler` 类 |
| 创建返回值 | `add_task()` 返回完整 task | 返回 `lastrowid` | 二次调用 `get_task()` 包装 |
| 更新方法名 | `update_task` | `update_status` | 使用实际方法名 |
| 更新返回值 | 返回 task 或 None | 无返回值 | 先 `get_task()` 查存在 → `update_status()` → 再 `get_task()` |
| 删除返回值 | 返回布尔 | 无返回值 | 先 `get_task()` 查存在 → `delete_task()` |
| 状态值域 | `pending/running/done/failed` | `todo/ready/running/done/blocked` | 校验扩展为 7 种状态全接受 |

**验证**：

| 测试 | 命令 | 结果 |
|------|------|:--:|
| 创建任务 | `POST /api/kanban {"title":"测试任务"}` | ✅ |
| 更新状态 | `PUT /api/kanban/38 {"status":"running"}` | ✅ |
| 删除任务 | `DELETE /api/kanban/38` | ✅ |
| 空标题 | `POST /api/kanban {}` | ✅ 400 |
| 不存在任务 | `PUT /api/kanban/99999` | ✅ 404 |
| 无效状态 | `PUT /api/kanban/38 {"status":"invalid"}` | ✅ 400 |

**踩坑**：
1. 路由模式差异：任务包假设 `app.router.add_post()`，但 zero 使用 aiohttp `RouteTableDef` + `@routes.view()` 装饰器。在 `KanbanHandler` View 类中新增 `post()` 方法处理 POST；新建 `KanbanTaskHandler` View 类（路由 `/api/kanban/{id}`）处理 PUT/DELETE。

3. `update_status` 方法名与任务包假设不同，且无返回值，需先查→更新→再查。
4. 状态值域不一致，校验扩展为 7 种全接受以兼容两套语义。

### 002 · 文件上传服务端校验

**目标**：防止恶意文件上传，增加类型/大小/路径穿越三重防护。

**实现**：

- `ALLOWED_EXTENSIONS`：50+ 白名单扩展名（文档/图片/音视频/代码/压缩包/字体）

- 路径穿越防护：`os.path.basename()` 剥离目录部分
- 目录安全：上传到 `data/uploads/` 隔离目录
