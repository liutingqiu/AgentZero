## 需求

将所有 API 错误响应格式统一为 `{ok: false, error: {code: string, msg: string}}`，替换当前混杂的 `{error: msg}`、`{reply: '处理失败...'}`、`'错误信息'` 等多种格式。

### 实现要点

1. **添加辅助函数（文件顶部，工具函数区域）**：

   ```python
   def _error_response(code: str, msg: str, status: int = 400) -> web.json_response:
       """统一的错误响应格式。"""
       return web.json_response(
           {'ok': False, 'error': {'code': code, 'msg': msg}},
           status=status
       )
   ```

2. **定义错误码常量（辅助函数旁边）**：

   ```python
   # 错误码常量
   ERR_AUTH_FAILED = 'AUTH_FAILED'
   ERR_FILE_TYPE = 'FILE_TYPE_NOT_ALLOWED'
   ERR_FILE_SIZE = 'FILE_TOO_LARGE'
   ERR_INTERNAL = 'INTERNAL_ERROR'
   ERR_BAD_REQUEST = 'BAD_REQUEST'
   ERR_NOT_FOUND = 'NOT_FOUND'
   ```

3. **替换所有错误返回**：

   搜索文件中所有返回错误的地方，格式替换规则：

   | 原格式 | 替换为 |
   |--------|--------|
   | `return web.json_response({'error': '...'}, status=400)` | `return _error_response(ERR_CODE, '...', status=400)` |
   | `return web.json_response({'reply': '处理失败...'})` | `return _error_response(ERR_INTERNAL, '处理失败...')` |
   | `return web.Response(text='...', status=500)` | `return _error_response(ERR_INTERNAL, '...', status=500)` |
   | `except Exception as exc: return web.json_response(...)` | `return _error_response(ERR_INTERNAL, str(exc), status=500)` |

   具体位置包括但不限于：
   - auth handler 中的认证失败和网络错误
   - upload handler 中的文件错误
   - collab handler 中的执行错误
   - 通用异常处理

4. **不要改动**：
   - 认证成功的 `200` 响应（如 `{'ok': True}`）
   - 健康检查的响应
   - SSE 流中的 event（`{type: 'error', data: ...}`）
   - Token 统计的响应

## 验收标准

1. 认证失败：`{"ok": false, "error": {"code": "AUTH_FAILED", "msg": "暗号错误"}}`
2. 上传非法文件类型：`{"ok": false, "error": {"code": "FILE_TYPE_NOT_ALLOWED", "msg": "文件类型不允许: .exe"}}`
3. 上传文件过大：`{"ok": false, "error": {"code": "FILE_TOO_LARGE", "msg": "文件过大，最大允许 10MB"}}`
4. 所有错误响应都包含 `ok`, `error`, `error.code`, `error.msg` 字段
5. 成功响应不受影响

## 如何验证

```bash
# 测试认证失败
curl -X POST -H "Content-Type: application/json" -d '{"code":"wrong"}' http://127.0.0.1:5052/api/auth

# 测试文件类型错误
echo "test" > /tmp/test.exe
curl -F "file=@/tmp/test.exe" http://127.0.0.1:5052/api/upload

# 确认成功响应不受影响
curl http://127.0.0.1:5052/health
```

预期输出：
```json
// 认证失败（400）
{"ok": false, "error": {"code": "AUTH_FAILED", "msg": "暗号错误"}}

// 健康检查（200）- 不变
{"status": "ok", "session": "已解锁"}
```

## 遇到问题时的决策树

1. 不确定某个响应是成功还是错误 → 看 status code，>= 400 是错误
2. 不确定某个错误应该用什么 error code → 停止，问清楚
3. 需要改其他文件才能完成 → 停止，报告
4. 发现两个错误路径的代码非常相似 → 只改格式，不重构
5. 发现现有代码有其他 bug → 记录但不修改，报告
